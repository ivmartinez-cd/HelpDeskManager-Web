# Antigravity Agent Rules

## MANDATORY WORKFLOW
For any task that is NOT a simple clarification or a trivial one-line fix:

1.  **ALWAYS invoke `@[brainstorming]` first.**
2.  Follow the brainstorming process strictly:
    -   Ask one question at a time.
    -   Identify and document all assumptions.
    -   Obtain an **"Understanding Lock"** (explicit user confirmation) before proposing a design or implementation plan.
3.  **NEVER** skip directly to implementation plans or code changes without the brainstorming phase.

## REASONING
The user values the **process** and **disciplined reasoning** as much as the final code. Rushing to a fix is considered a failure of the methodology.

## SKILL USAGE
- Use `@[skill-name]` to reference skills.
- Read `SKILL.md` files before executing specialized tasks.
